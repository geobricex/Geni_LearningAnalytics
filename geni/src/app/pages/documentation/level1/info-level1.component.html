Level 1 contains the structured representation of individual academic data, captured from virtual classroom educational platforms. It is designed to facilitate interoperability between consumer applications and educational analytics web services, allowing automated processing, descriptive analysis, visualization generation, and interpretation return using language models.

Level 1 is an array of objects, where each object corresponds to a student enrolled in a specific course, with a set of associated academic activities. This hierarchical structure is compatible with object-oriented analysis schemas and DataFrame-like structures.
  • student_id (str): Unique identifier of the student.
  • student_name (str): Full name of the registered student.
  • course_id (str): Course identifier code.
  • course_name (str): Formal name of the course.
  • activities (list): List of activities evaluated by the teacher.
  • activity_id (str): Unique identifier of the activity.
  • type (str): Type of academic activity ("quiz", "forum", "task", etc.).
  • title (str): Name or description of the activity.
  • grade (float): Grade obtained with up to two decimals.
  • max_grade (float): Maximum score set for the activity.
  • submission_date (str, ISO 8601): Submission date and time in `YYYY-MM-DDTHH:MM:SSZ` format (UTC).
  • observation (str): Teacher's qualitative comment on the student's performance.
